🔐 2. Authentication & Security
2.1 Credentials

You need the following:

Client ID → Found in your app settings
API Key → Shown once during creation (store securely)
JWT Token → Used for API requests (expires in 1 hour)
🔁 How Authentication Works
Use Client ID + API Key
Exchange them for a JWT Token
Use the JWT Token in all API requests
Refresh token after 60 minutes
2.2 Security Best Practices
Never expose API keys publicly
Always use HTTPS
Store credentials securely (e.g., environment variables)
2.3 Rate Limits
120 requests per minute per IP
If exceeded → requests are throttled

👉 Recommended:

Implement retry logic with exponential backoff

🔔 5. Webhooks (Real-Time Updates)
5.1 What Are Webhooks?

Webhooks are HTTP POST callbacks sent when events happen (e.g., payment success).

5.2 Webhook Levels
Level	Scope	Usage
Merchant	All transactions	Default
Application	API transactions only	Recommended for API apps

👉 If app webhook exists → it overrides merchant webhook

5.3 Supported Events
PAYMENT RECEIVED
PAYMENT FAILED
PAYOUT INITIATED
PAYOUT REFUNDED
PAYOUT REVERSED
DEPOSIT RECEIVED (merchant only)
5.4 Example Webhook Payload
{
  "event": "PAYMENT RECEIVED",
  "data": {
    "id": "ORD123",
    "status": "SUCCESS",
    "collectedAmount": "10000",
    "collectedCurrency": "TZS",
    "customer": {
      "customerName": "John Doe",
      "customerEmail": "john@example.com"
    }
  }
}
5.5 Setting Up Webhooks
Merchant-Level
Go to Settings → Developers
Find Webhooks
Add URL per event
Application-Level
Open your application
Add webhook URLs per event
5.6 Handling Webhooks
ClickPesa sends HTTP POST
Your server must return HTTP 2xx

⚠️ Important:

2xx = received (NOT processed)
Validate payload using checksum

🔑 7. Checksum (Data Integrity)
Why It Matters

Ensures:

Requests are not tampered
Webhooks are authentic
⚠️ Important Rules
Exclude:
checksum
checksumMethod
Regenerate tokens after enabling/disabling checksum
7.1 How It Works
Sort payload keys (alphabetically)
Convert to JSON string
Generate HMAC-SHA256
Return hex string
Example (JavaScript)
const crypto = require("crypto");

function createChecksum(key, payload) {
  const sorted = JSON.stringify(payload);
  return crypto.createHmac("sha256", key)
               .update(sorted)
               .digest("hex");
}
7.2 Validation
Remove checksum fields
Recompute checksum
Compare values
Match → valid
Not match → reject
